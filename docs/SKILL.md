# SKILL.md — Stock-Ops 스킬 정의

## 이 파일의 역할

Claude Code가 `/annotate` 명령어를 받았을 때 어떻게 동작할지를 정의한다.
각 모드는 독립적으로 실행된다. 필요한 컨텍스트만 로드한다.

---

## 공통 컨텍스트 로딩 순서

모든 모드 실행 전 다음 순서로 읽는다.

1. `CLAUDE.md` — 행동 원칙 확인
2. `config/profile.yml` — 사용자 투자 성향
3. `data/trades.json` — 매매 이력 전체
4. 해당 모드에 필요한 추가 데이터

---

## MODE: analyze

### 목적
특정 매매 1건을 기술적으로 분석하고, 당시 매수 근거와 실제 시장 상황을 대조한다.

### 입력
- `trade_id`: 분석할 매매의 id

### 실행 순서

**Step 1. 매매 데이터 로드**
```
trades.json에서 해당 id의 레코드를 찾는다.
ticker, date, price, action, reason을 확인한다.
```

**Step 2. 시장 데이터 수집**
```
market_cache/{ticker}/{date}_ohlcv.json이 있으면 그대로 사용.
없으면 Yahoo Finance API로 매수일 기준 전5일~후5일 데이터를 가져온다.
캐시에 저장한다.
```

**Step 3. 기술적 지표 계산**
```python
# pandas-ta 사용
df['RSI'] = ta.rsi(df['close'], length=14)
df['volume_ratio'] = df['volume'] / df['volume'].rolling(5).mean()
df['pct_change'] = df['close'].pct_change() * 100
```

매수 당일 기준으로 다음을 추출한다:
- RSI 값
- 전일 대비 등락률
- 거래량 비율 (5일 평균 대비)
- 5일 전 대비 누적 상승률 (매수 전 모멘텀)

**Step 4. 기술적 상태 평가**

다음 기준으로 신호를 판단한다:

| 지표 | 주의 신호 | 양호 신호 |
|---|---|---|
| RSI | 70 이상 (과매수) | 40~60 |
| 전일 등락률 | +3% 이상 (추격 매수 위험) | +1% 이하 |
| 거래량 비율 | 3배 이상 (급등 후 피크 가능성) | 1~1.5배 |
| 5일 누적 | +10% 이상 (단기 고점 가능성) | +5% 이하 |

**Step 5. reason 대조 분석**

사용자가 입력한 reason과 실제 시장 상황을 대조한다.

분석 포인트:
- reason에 근거가 있었는가? (뉴스, 지표, 섹터 이슈 등)
- 근거와 실제 시장 상황이 일치했는가?
- 매수 타이밍은 적절했는가?

**Step 6. 결과 이후 추적** (매도 완료 건만)

- 매수가 대비 최종 손익
- 매도 시점의 기술적 상태
- 실제 결과와 당시 판단의 정합성

**Step 7. 리포트 저장**

`reports/{id}-{ticker}-{date}.md` 형식으로 저장.

### 출력 형식

```
## [삼성전자 005930] 매수 분석 — 2025-04-10

### 매수 시점 기술적 상태
- RSI: 74.2 ⚠️ (과매수 구간)
- 전일 대비: +3.8% ⚠️ (급등 후 추격 매수)
- 거래량: 평균 대비 2.4배
- 5일 누적 상승: +8.2%

**종합 평가**: 매수 시점이 단기 고점에 가까웠을 가능성이 있습니다.
RSI와 당일 급등폭이 모두 주의 구간에 해당합니다.

### 매수 근거 vs 시장 상황
당신의 매수 근거: "AI 반도체 수요 증가, HBM 수혜 기대"
당시 시장 상황: 섹터 뉴스는 긍정적이었으나, 단기 급등 후 매수 타이밍.
→ 근거 자체는 유효했으나, 타이밍 선택에 개선 여지가 있습니다.

### 결과 (매도 완료)
매도가: 75,000원 | 수익률: +4.2% | 익절 ✅
```

## MODE: review

### 목적
전체 매매 이력을 종합 분석하여 나의 매매 습관 패턴을 도출한다.
데이터가 많을수록 정밀도가 올라간다.

### 실행 순서

**Step 1. 전체 이력 로드**
`trades.json` 전체를 읽는다. buy 건수, sell 건수, 현재 보유 건수를 파악한다.

**Step 2. 매수 습관 분석**

다음 항목을 집계한다:
- 매수 시 RSI 분포 (평균, 70 이상 비율)
- 매수 당일 전일 대비 등락률 분포
- 매수 시 거래량 비율 분포
- 요일별 매수 빈도
- reason 텍스트 키워드 분석 (근거 있음 / 감에 의존 비율)

**Step 3. 익절/손절 패턴 분석**

closed / loss 건을 대상으로:
- 평균 보유 기간
- 익절 시 수익률 분포
- 손절 시 손실률 분포
- 익절 케이스의 매수 시 RSI 평균
- 손절 케이스의 매수 시 RSI 평균

**Step 4. 패턴 문장 생성**

다음 형식으로 정리한다:

```
패턴상 당신은 ---합니다.
[N건 중 N건에서 확인됨]
```

최소 3개, 최대 7개의 패턴 문장을 도출한다.

**Step 5. 리포트 저장**
`reports/pattern-{date}.md`에 저장.

### 출력 형식

```
## 매매 습관 패턴 분석 — 2025-04-14
분석 대상: 총 12건 (매수 12, 매도 8, 보유 4)

### 패턴 요약

패턴상 당신은 전일 급등일에 매수하는 경향이 있습니다.
[12건 중 8건에서 매수 당일 전일 대비 +2% 이상 확인]

패턴상 당신의 매수 시 RSI 평균은 68.3으로, 과매수 구간에서 진입하는 경우가 잦습니다.
[RSI 70 이상 매수: 12건 중 5건]

패턴상 익절한 케이스(6건)의 매수 시 RSI 평균은 58.1이었고,
손절한 케이스(2건)의 매수 시 RSI 평균은 76.4였습니다.
→ RSI가 낮을 때 매수한 경우에서 수익이 났을 가능성이 높습니다.

패턴상 매수 근거를 구체적으로 적은 경우(7건)의 수익률 평균은 +6.2%,
"그냥", "좋아 보여서" 등 막연한 근거인 경우(5건)는 -2.1%입니다.

### 개선 포인트
1. 급등 당일보다 눌림 후 매수 타이밍 고려
2. RSI 65 이상 신규 매수 재검토
3. 매수 전 근거를 구체적으로 메모하는 습관
```

## MODE: brief

### 목적
오늘 보유 중인 종목들의 최신 뉴스를 3줄 요약으로 제공한다.

### 실행 순서
1. `status: holding`인 종목 리스트 추출
2. 각 종목별 최신 뉴스 3건 수집
3. 종목당 3줄 이내로 요약
4. `news_cache/{ticker}/{date}.json`에 저장

### 출력 형식
```
## 보유 종목 브리핑 — 2025-04-14

### 삼성전자 (005930)
HBM3E 퀄컴 납품 계약 소식. 2분기 실적 기대감 상승.
외국인 순매수 3일 연속. 기관은 소폭 매도 중.

### 카카오 (035720)
AI 사업부 분리 검토 보도. 주가 반응은 제한적.
```

## MODE: why

### 목적
특정 매매를 다시 꺼내어, 당시 왜 그 결정을 했는지 돌아보게 한다.

### 실행 순서
1. 해당 매매의 reason 필드 로드
2. 당시 시장 상황 (이미 캐시된 데이터 활용)
3. 근거 평가: 있음 / 약함 / 없음
4. 결과와 대조하여 피드백

### 출력 형식
```
## [삼성전자] 2025-04-10 매수 — 왜 샀었나요?

당신의 기록: "AI 반도체 수요 증가, HBM 수혜 기대. 뉴스는 따로 안 봤고 그냥 요즘 AI 반도체 얘기가 많아서."

근거 평가: 약함
→ 섹터 방향성은 맞았지만, 구체적인 근거(실적, 수주, 뉴스)보다는
  분위기에 의존한 매수였습니다.

결과: +4.2% 익절 ✅
→ 이번엔 수익이 났지만, 같은 방식으로 반복되면
  언제든 반대 결과가 나올 수 있습니다.
```
