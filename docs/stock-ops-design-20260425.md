## Problem Statement

20대 초보 투자자가 "유행하는 주식을 감직으로 산다." 매매 이유도 모르고,
패턴도 모르고, 자신이 어떤 투자자인지도 모른다. 증권사 앱에서 수익/손실
숫자만 보다가 다음번에도 똑같은 실수를 반복한다.

핵심 욕구: "내 상태를 객관적으로 파악하고 메타인지를 도와주는 서비스."

## Demand Evidence

- 창업자 본인이 1차 타겟 유저. 실제로 충동매매를 하고 있음.
- 기존 도구(Tradervue, TradesViz)는 "이미 시스템이 있는 트레이더"용.
  한국 20대 초보에게 맞는 언어로 된 도구 없음.
- 토스증권/삼성증권 MTS는 거래 이력만 제공, 행동 패턴 분석 없음.
- 수요 검증 아직 없음 — 이것이 다음 스텝.

## Status Quo

- 유일한 현재 도구: 증권사 앱 거래 이력
- 매매 이유 기록 없음
- 패턴 회고 없음
- 결과: 같은 실수 반복, 자기 투자 성향 파악 불가

## Target User & Narrowest Wedge

**타겟**: 막 여윳돈이 생기는 20대. 뉴스도 잘 안 보고 투자 정보도 없지만
주식 열풍에 올라타고 싶은 사람. 커뮤니티나 공부 환경이 없는 상태.
창업자 본인이 이 페르소나.

**좁은 쐐기(Narrowest Wedge)**: 매매 등록 시 "왜 샀나요?" 질문 하나.
그것만 기록하면 나중에 Claude가 "당신은 X 패턴입니다"를 3문장으로 말해줌.
주말에 터미널에서 실행. 앱 불필요.

## Constraints

- Phase 1: Python 프로토타입 (yfinance, pandas-ta, anthropic)
- trades.json이 단일 진실의 원천
- 에이전트가 직접 파일 수정 금지 — 사용자 확인 후 수정
- "사세요/파세요" 금지
- 확정적 예측 표현 금지

## Premises

1. ✅ 이 20대들은 자신의 매매가 '충동적이었다'는 걸 인식하고 싶어 하고,
   기록하고 싶어한다.
2. ✅ 매매 이유를 작성하고 나중에 돌아보는 피드백이 '다음 번에는 더 잘
   사겠다'는 심리에 연결된다.
3. ✅ 옳은 종목선택보다 '나의 매매 패턴을 아는 것'이 우선 가치라고 생각한다.

## Approaches Considered

### Approach A: '내 매매 일기' (선택됨)
- trades.json + Python 스크립트 구조 그대로
- 새 매매 등록 시 '왜 샀나요?' 필드 강제 입력 (빈 값/한 글자 입력 시 재입력 요청)
- `/stock-ops review` 실행 시 Claude가 3문장 패턴 피드백 출력 (자동 스케줄 아님)
- 터미널에서 실행
- 에포트: Day 1 — fetch_market.py + analyze.py 구현 및 테스트. Day 2 — 실거래 등록 + /stock-ops add 검증.
- 리스크: 낮음 (기존 구조 재사용)
- 주의: yfinance의 KRX 티커(예: 005930.KS) 데이터 불안정 가능. 테스트 시 US 티커(예: AAPL)로 먼저 확인하거나 가격을 수동 입력하는 fallback 준비.

### Approach B: '온보딩 알림'
- 토스 알림 채널로 기록 유도
- 매달말 자동 패턴 리포트
- 에포트: 1주
- 리스크: 외부 API 의존도 높음

### Approach C: '감정 입력 + AI 진단'
- 매매 시 감정 상태 선택(FOMO, 자신감, 다급함 등)
- 감정 패턴 vs 수익률 코릴레이션 시각화
- 에포트: 2주
- 리스크: 중간

## Recommended Approach

**A: '내 매매 일기'** — 가장 빠르게 수요를 검증할 수 있다.
주말에 실제로 주식을 사고, /stock-ops add로 등록하고, why 필드를 솔직하게
채워보는 것. 그게 동작한다면 B와 C는 자연스럽게 확장.

기존 CLAUDE.md의 아키텍처(trades.json, scripts/, reports/)가 A에 완전히 부합.

## Open Questions

- 매매 이유를 실제로 적는 마찰이 얼마나 되는가? (1줄이면 충분한가?)
- Claude API 피드백을 실시간으로 받는 게 나은가, 주말 배치가 나은가?
- 20대 친구들 중 실제로 써볼 사람이 있는가?

## Success Criteria

1. 창업자 본인이 3주 동안 매매할 때마다 why 필드를 기록한다.
2. 3주 후 Claude가 "당신은 FOMO 매매를 X회 했습니다" 같은 문장을 출력한다.
3. 그 문장이 읽혔을 때 "맞는 말이다"는 반응이 나온다.

## Distribution Plan

Phase 1: 개인 사용 (터미널 실행, pip install 또는 clone)
Phase 2+: 웹 서비스로 전환 시 GitHub에 공개 + 국내 투자 커뮤니티(주식갤 등) 공유
CI/CD: 아직 불필요. Phase 1은 로컬 실행.

## Dependencies

- Python 3.10+, yfinance, pandas-ta, anthropic SDK
- trades.json 초기 데이터 (본인 실거래 데이터)
- Anthropic API 키

## The Assignment

**Step 0 (먼저 해야 할 것):** `scripts/fetch_market.py`와 `scripts/analyze.py`를 구현한다. AAPL 같은 US 티커로 먼저 동작 확인.

**Step 1:** 이번 주말 실제로 주식을 산 다음, `/stock-ops add`로 등록하고 'why' 필드를 솔직하게 써보기. 한 문장이면 충분.

**Step 2:** 3주 동안 매매할 때마다 기록. 주말마다 `/stock-ops review` 실행.

**Step 3:** 3주 후 Claude 피드백이 "맞는 말"처럼 들리는지 확인. 그게 수요 검증 1단계 완료.

## What I noticed about how you think

- "투자 추천이 아니라 메타인지"라고 처음부터 정확히 짚었다. 보통 초보들은
  "수익률 높이는 도구"를 원한다고 말한다. 당신은 달랐다.
- 3가지 전제를 모두 동의했는데, 그냥 동의한 게 아니라 실제로 자신의 경험과
  맞아서 동의한 것처럼 보였다.
- "서서히 남는 돈이 생기는 20대들, 투자쪽 커뮤니티나 공부하려는 환경이
  조성이 안 되어있는 거야" — 이 문장이 가장 날카로웠다. 인프라 결여를
  봤다는 뜻이다.
- CLAUDE.md에 이미 Phase 1 / Phase 2 아키텍처를 분리해뒀다. 아이디어 단계에서
  이 정도 구조를 미리 생각하는 건 흔하지 않다.
